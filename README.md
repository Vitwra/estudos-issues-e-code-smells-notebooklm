# estudos-issues-e-code-smells-notebooklm

## 🎯 Contexto e Objetivos
Este estudo de caso busca apresentar introdutoriamente o conceito de **Issues** e **Code Smells** em sistemas de software. Aborda conhecimentos básicos e implicações desses desafios durante as fases de teste, ajuste e refatoração. Além disso, busca explorar e analisar a necessidade do tratamento e como agir corretamente para tornar o código limpo e funcional.

O NotebookLM atua como um **professor especialista** na área de engenharia de software e tem como um dos objetivos ser educativo e cuidadoso na abordagem do tema.

## 📖 Miniguia de Estudo

### Prompts Reutilizáveis:
Você pode utilizar os prompts abaixo para revisar o conteúdo ou aprofundar os estudos:

1.  **O que são code smells e issues na engenharia de software?** Existem meios para facilitar a identificação e resolução desses problemas?
2.  **Explique os cinco componentes fundamentais** para desenvolver uma mentalidade orientada a soluções.
3.  **Como a refatoração pode ser um auxílio** na manutenção e solução desses issues e code smells?
4.  **Quais outras técnicas são populares no meio** para garantir a qualidade do código?

### Glossário de Conceitos Aprendidos:
* **Code Smells:** Sinais superficiais que geralmente indicam um problema mais profundo no design do sistema.
* **Issues:** Registros de problemas, falhas ou melhorias necessárias em um projeto de software.
* **Refatoração:** Técnica de reestruturação do código para melhorar sua estrutura interna sem alterar seu comportamento externo.
* **Clean Code (Código Limpo):** Práticas de programação que tornam o código fácil de ler, entender e manter.
